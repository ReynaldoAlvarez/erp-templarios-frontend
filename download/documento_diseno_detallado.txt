2. ARQUITECTURA TÉCNICA DEL SISTEMA
2.1 Visión General de la Arquitectura
El sistema ERP TEMPLARIOS se implementa con una arquitectura de dos componentes principales separados: BACKEND (API
RESTful): Servidor Express.js independiente que expone APIs RESTful, maneja la lógica de negocio, autenticación, autorización, y
acceso a datos. Se despliega en Digital Ocean como servicio independiente. FRONTEND (PWA): Aplicación Progressive Web App
desarrollada con Next.js que consume las APIs del backend. Funciona como aplicación web tradicional y puede instalarse como app
nativa en dispositivos móviles. Incluye soporte offline mediante Service Workers. Esta separación permite: Escalar frontend y
backend independientemente, desarrollar y desplegar cada parte por separado, usar el backend para futuras integraciones (app móvil
nativa, integraciones externas), y mantener la seguridad de la lógica de negocio en el servidor.

                            Entidad                              Relaciones Principales                           Tipo

                  Role                   belongsToMany: User, Permission                                  Seguridad

                  Permission             belongsToMany: Role                                              Seguridad

                  Empleado               belongsTo: Empresa, Sucursal; hasOne: Conductor                  RRHH

4. API RESTful - ENDPOINTS COMPLETOS
4.1 Módulo IAM - Autenticación y Autorización
         Método                     Endpoint                                 Descripción                              Permisos

          POST     /api/auth/login                    Autenticar usuario, retorna access + refresh tokens   Público

          POST     /api/auth/logout                   Invalidar sesión y refresh token                      Autenticado

          POST     /api/auth/refresh                  Renovar access token con refresh token                Refresh token válido

          POST     /api/auth/forgot-password          Enviar email de recuperación                          Público

          POST     /api/auth/reset-password           Establecer nueva contraseña                           Token válido

          GET      /api/users                         Listar usuarios paginado                              iam:usuarios:listar

          GET      /api/users/:id                     Obtener usuario por ID                                iam:usuarios:leer

          POST     /api/users                         Crear usuario (envía email con password)              iam:usuarios:crear

          PUT      /api/users/:id                     Actualizar usuario                                    iam:usuarios:actualizar

         DELETE    /api/users/:id                     Desactivar usuario (soft delete)                      iam:usuarios:eliminar

          GET      /api/roles                         Listar todos los roles                                iam:roles:listar

5. INTERFACES DE USUARIO (WIREFRAMES DESCRIPTIVOS)
5.1 Dashboard Principal
El dashboard principal será la pantalla de inicio después del login, mostrando información crítica de un vistazo: ENCABEZADO:
Logo empresa, nombre usuario, avatar, notificaciones (badge con contador), botón logout. BARRA LATERAL (Sidebar):
Navegación principal colapsable con iconos y labels. Secciones: Dashboard, Operaciones (BL, Viajes), Flota (Camiones,
Conductores, Trailers), Documentos, Finanzas (Facturas, Liquidaciones, Activos/Pasivos), Reportes, Administración (Usuarios,
Roles, Empresa). Footer con versión del sistema y estado de conexión (online/offline). ÁREA PRINCIPAL - TARJETAS KPI: 4
tarjetas en fila mostrando: Viajes Activos (con badge de estado), Camiones Disponibles (verde)/En Ruta (azul)/Mantenimiento
(naranja), Documentos Pendientes (con alerta si > 5), Facturación del Mes (con comparación vs mes anterior). GRÁFICOS: Fila de 2
gráficos - Gráfico de líneas: Viajes últimos 30 días. Gráfico de barras: Rentabilidad por camión (top 5). Fila inferior - Tabla: Últimos
5 viajes con estado (cards con indicador de color según estado). Lista de alertas: Documentos pendientes, licencias por vencer, pagos
pendientes. MODO OFFLINE: Cuando no hay conexión, mostrar indicador visual en el header. Los datos mostrados serán los
últimos sincronizados. Los cambios se guardarán localmente y se sincronizarán cuando vuelva la conexión.


5.2 Módulo de Usuarios y Roles
LISTADO DE USUARIOS: Tabla con columnas: Avatar, Nombre, Email, Roles (badges de colores), Estado (Activo/Inactivo
badge), Último acceso, Acciones (Ver, Editar, Desactivar). Filtros: Por rol, por estado, búsqueda por nombre/email. Botón 'Nuevo
Usuario' arriba a la derecha. FORMULARIO CREAR USUARIO: Campos: Email (único, validación), Nombre, Apellido, Teléfono
(opcional), Sucursal (dropdown), Roles (multiselect con chips), Foto (opcional, upload). Al crear: Generar contraseña automática,
enviar email con credenciales, marcar emailVerificado = false. Validación: Email único, al menos un rol. GESTIÓN DE ROLES:
Vista de cards con cada rol mostrando: Nombre, Descripción, Color (preview), Número de usuarios asignados, Número de permisos.
Botón 'Nuevo Rol' para crear roles personalizados. Modal de edición con: Nombre, Descripción, Color picker, Lista de permisos por
módulo (checkboxes agrupados). PERMISOS EFECTIVOS: Vista de usuario mostrando todos los permisos heredados de sus roles,
agrupados por módulo con indicador de qué rol otorga cada permiso.


5.3 Módulo de Operaciones (BL y Viajes)
LISTADO DE BL: Tabla con columnas: Número BL, Cliente, Peso Total, Unidades, Origen, Destino, Estado (badge color), # Viajes,
Acciones. Filtros: Por cliente, por estado, por fecha, búsqueda por número. Botón 'Nuevo BL' y 'Importar desde Excel'. Card de
resumen: Total BLs pendientes, en proceso, completados. FORMULARIO BL: Secciones: Datos del BL (Número único, Cliente
dropdown, Tipo entrega), Datos de Carga (Peso total, Unidades, Tipo de carga), Ruta (Puerto origen, Aduana, Destino final, Nave),
Estado. Botón 'Calcular Flota' que muestra cuántos camiones se necesitan según capacidad. PROGRAMACIÓN DE VIAJES: Desde
un BL, botón 'Programar Viaje' abre modal con: Camión (dropdown de disponibles), Conductor (dropdown de disponibles), Fecha
salida, Peso asignado, Generar MIC automático o manual. Tabla de viajes programados para el BL con estado de cada uno.
SEGUIMIENTO DE VIAJES: Timeline vertical mostrando: Salida de puerto → Cruces de frontera (con canal y fechas) → Llegada a
destino. Cada paso con fecha/hora, estado, y documentos asociados. Formulario para registrar cambios de canal en fronteras (nunca
edita, siempre inserta nuevo registro). DETALLE DE VIAJE: Tabs: General (datos principales), Tramos, Fronteras (historial de
canales), Documentos (lista con estado), Liquidación (cálculos financieros). Indicadores de estado con colores: Programado (gris), En
Ruta (azul), En Aduana (amarillo), Entregado (verde), Cancelado (rojo).


5.4 Módulo de Flota
LISTADO DE CAMIONES: Vista de tarjetas o tabla con: Foto/Icono, Placa, Marca/Modelo, Capacidad (Tn), Estado (badge color),
Disponibilidad, Kilometraje, Acciones. Filtros: Por estado, por sucursal, por capacidad. Indicador visual de documentación por
vencer (licencia, tarjeta operación). DETALLE DE CAMIÓN: Tabs: Información general, Documentación (tarjeta operación con
vencimiento y alerta), Viajes (histórico), Mantenimientos (historial y próximos), Rentabilidad (gráfico de ingresos vs gastos).
Métricas: Viajes totales, Ingresos totales, Gastos totales, Rentabilidad neta. LISTADO DE CONDUCTORES: Tabla con: Foto,
Nombre, Licencia, Vencimiento licencia (alerta si próximo), Disponibilidad, Calificación, # Viajes, Acciones. Filtros: Por
disponibilidad, por tipo contrato, por sucursal. DETALLE DE CONDUCTOR: Tabs: Información personal, Historial de viajes,
Gastos (tabla con totales), Sanciones/Incidentes (historial), Rentabilidad (ingresos generados vs pagos). Cards de resumen: Total
viajes, Total ingresos generados, Calificación promedio, Gastos totales. REGISTRO DE GASTOS: Modal con: Conductor
(autoseleccionado si desde detalle), Viaje (opcional), Concepto, Monto, Categoría, Fecha, Comprobante (foto opcional),
Observaciones. Lista de gastos con totales por categoría y período.


5.5 Módulo de Reportes
DASHBOARD DE REPORTES: Grid de tarjetas con acceso a cada reporte. Filtros globales de fecha y sucursal aplicables a todos los
reportes. REPORTE RENTABILIDAD MENSUAL: Gráfico de líneas con ingresos, gastos y utilidad neta por mes. Tabla con detalle
mensual exportable. Comparación con año anterior (opcional). REPORTE RENTABILIDAD POR VEHÍCULO: Tabla ordenada por
rentabilidad con: Placa, # Viajes, Ingresos, Gastos (combustible, mantenimiento, otros), Utilidad neta, % Utilidad. Gráfico de barras
horizontal. REPORTE TOP CLIENTES: Tabla con: Cliente, # BLs, # Viajes, Facturación total, % del total. Gráfico de torta con
distribución. REPORTE FLUJO DE CAJA: Tabla tipo cash flow con: Fecha, Concepto, Entradas, Salidas, Saldo. Gráfico de área
acumulativa. Filtros por período. REPORTE GASTOS POR CATEGORÍA: Gráfico de torta con distribución. Tabla con totales por
categoría y % del total. Drill-down a detalle de gastos. REPORTE DEMORAS EN ADUANAS: Tabla con: Frontera, Canal, Tiempo
promedio, # de casos, Impacto en días. Gráfico de barras por tipo de canal. Identificación de patrones de demora. EXPORTACIÓN:
Todos los reportes tienen botón de exportar a Excel o PDF. El Excel mantiene formato de los registros actuales para familiaridad.

DOCUMENTOS LOCALES: Opción de guardar documentos localmente sin subir a nube. Estructura de carpetas por:

8. CRITERIOS DE ACEPTACIÓN POR MÓDULO
8.1 Módulo IAM
AUTENTICACIÓN: Usuario puede iniciar sesión con email y contraseña válidos. Token de acceso se genera con expiración de 15
minutos. Refresh token se genera con expiración de 7 días y se persiste en BD. Usuario bloqueado después de 5 intentos fallidos.
Email de recuperación de contraseña funcional. Logout invalida sesión y refresh token. GESTIÓN DE USUARIOS: Administrador
puede crear usuarios con email único. Contraseña generada automáticamente y enviada por email. Usuario debe cambiar contraseña
en primer login. Administrador puede editar datos de usuario. Administrador puede desactivar usuario (soft delete). Usuario ve su
propio perfil y puede editar datos básicos. ROLES Y PERMISOS: Administrador puede crear roles personalizados. Administrador
puede asignar múltiples roles a un usuario. Los permisos efectivos son la unión de todos los roles. Usuario sin permiso específico no
puede acceder al recurso. API rechaza peticiones sin permisos adecuados (403). Auditoría registra quién otorgó cada rol.


8.2 Módulo BL y Carga
GESTIÓN DE CLIENTES: CRUD completo de clientes con validación de NIT único. Búsqueda de clientes por nombre o NIT.
GESTIÓN DE BL: Crear BL con datos completos del documento. Número de BL único validado. Cálculo automático de camiones
necesarios según peso y capacidad. Filtro de BLs por cliente, estado, fecha. Importación de BLs desde Excel (formato definido).
Exportación de BLs a Excel. APROBACIONES: Creación de BL nuevo requiere aprobación de Gerente/Administrador. Notificación
por email al aprobador. Registro de aprobación en auditoría.


8.3 Módulo Flota
GESTIÓN DE CAMIONES: CRUD completo de camiones. Placa única validada. Asignación de trailer a camión. Historial de viajes
por camión. Registro de mantenimientos. Alertas de vencimiento de tarjeta de operación. Cálculo de rentabilidad por camión
(ingresos - gastos). GESTIÓN DE CONDUCTORES: CRUD completo de conductores. Licencia única validada. Alertas de
vencimiento de licencia. Historial de viajes, sanciones, gastos. Cálculo de rentabilidad por conductor. Control de disponibilidad.
REGISTRO DE GASTOS: Registrar gastos por conductor. Asociar gasto a viaje (opcional). Categorización de gastos. Totales por
categoría y período.


8.4 Módulo Viajes
PROGRAMACIÓN: Crear viaje asociado a un BL. Asignar camión y conductor disponible. Generar MIC/DTA automático o manual.
Validar que camión no esté en otro viaje activo. Validar que conductor esté disponible. GESTIÓN DE FRONTERAS: Registrar cruce
de frontera con fecha/hora. Registrar canal asignado (Verde/Amarillo/Rojo). Cambios de canal crean nuevo registro (nunca editan).
Historial completo de canales por frontera. Justificación de retrasos. DOCUMENTOS: Subir documentos por viaje (MIC, CRT,
ASPB, etc.). Marcar documento como recibido. Lista de documentos pendientes. Almacenamiento local opcional. ESTADOS: Flujo
de estados: Programado → En Ruta → Aduana → Entregado. Cancelación con motivo obligatorio. Registro de cambios de estado en
auditoría.


8.5 Módulo Finanzas
LIQUIDACIONES: Crear liquidación por viaje con cálculo automático. Fórmula: PagoLiquido = Flete - IT 3% - Factura 7% -
Comisión - Anticipo. Aprobación de Gerente/Finanzas para pagar. Registro de fecha y responsable de pago. Historial de
liquidaciones. FACTURACIÓN: Crear factura asociada a uno o más viajes. Número de factura único. Estados: Pendiente → Emitida
→ Pagada / Anulada. Exportación en formato compatible con SIN. Aprobación de Gerente/Administrador para emitir. ACTIVOS Y
PASIVOS: CRUD completo de activos. CRUD completo de pasivos. Cálculo de depreciación (opcional). Estados de pago de pasivos.
